Assertive, Documentación Api

Por favor solicita URL, usuario y contraseña a tu representante de TI.

Todas las peticiones son encryptadas con un certificado válido para la url y se utiliza un puerto que por lo general es el 8443, así que las peticiones las deberás hacer por https://url:8443/

La llave "key" devuelta por el endpoint login, es tu permiso de acceso, no la compartas ni la pierdas o tu información se verá comprometida. La debes enviar con cada petición de cualquier método, ya sea en una cabecera de nombre key, o en el cuerpo del mensaje. ej. GET:

https://miserv.assertivebusiness.com:8443/endpoint?key=MiLlave

Se aceptan formatos estandar form-data x-www-form-urlencoded o json. TODOS los parámetros no esperados son IGNORADOS

Las respuestas son de tipo application/json y manejan los codigos apropiados para cada una. Para las de error el código esta fuera del rango de los 200 ej. 403 {"error":"No autorizado"}
